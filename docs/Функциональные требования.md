# Функциональные требования

## Интерактивный музейный стенд ГрГУ

**Проект:** `museum` (monorepo full-stack TypeScript)  
**Версия документа:** 1.0  
**Основание:** исходный код репозитория, схема БД (`schema.prisma`), маршруты API, клиентские маршруты (`routes.tsx`), компоненты админ-панели  
**Статус:** описывает фактически реализованную функциональность

---

## Обозначения

| Элемент | Описание |
|---------|----------|
| **ID** | Уникальный идентификатор требования |
| **Посетитель** | Пользователь публичного стенда без аутентификации |
| **Администратор** | Авторизованный пользователь админ-панели (`/admin`) |
| **Система** | Программный комплекс `@museum/web` + `@museum/server` |

Нумерация требований: **ФТ-{группа}-{номер}**, где группа: `П` — посетители, `А` — администраторы, `К` — управление контентом, `Н` — поиск и навигация, `С` — системные функции.
